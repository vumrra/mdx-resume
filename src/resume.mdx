## [GOGO](https://github.com/team-gogo) (2025. 01 ~ Now)

GOGO는 전국 중·고등학교 대상 스포츠 행사 플랫폼으로, 쉽고 즐겁게 경기 운영을 돕는 서비스입니다.  
본교 체육대회에서 <span class = "bold">+350명 DAU</span>를 기록하며 서비스 중이고, 전국 단위 확장을 목표로 발전하고 있습니다.  
전반적인 Restful API 개발과 성능 개선 작업을 진행하였고, (V2 → V3) 마이크로서비스 전환을 주도했습니다.

<span class="bold">
  제품 기획, 요구사항 분석, 서버 구축 및 퍼포먼스 향상 작업 진행
</span>

- V2 → V3 리뉴얼로 <span class = "bold">전국 확장이라는 OKR 목표</span>를 세워, 기획과 요구사항 명세 작업을 진행
  - 주요 사용자 대상 설문 조사를 통해 문제를 정의하고, 이를 바탕으로 개선 방향을 수립
  - 청소년 대상 불법 도박 중독 문제를 해결을 위한 기능 및 캠패인 요소 기획
    - 본교 운영 기간 동안 200명 이상의 학생들이 도박 예방 캠패인에 자발적으로 참여하도록 유도
  - 교내 전시회에 출품하여 2위 수상, 교육청 관계자로부터 실제 도입 제안, 신사업 제안과 피드백을 받음
- 전시 API의 응답 시간을 개선했으며, 비동기 큐(Kafka)를 활용하여 처리량을 개선
  - Redis Zset, 커버링 인덱스를 적용해 랭킹 조회 API의 응답속도를 <span class = "bold">1.3s → 0.3s로 200%</span> 개선
  - 복잡한 처리가 필요한 API들을 비동기 이벤트 기반으로 구성해, <span class = "bold">전체 API의 레이턴시를 0.2s 이하</span>로 개선
- 배팅 정산 처리 Spring Batch 로직을 개선하여, <span class = "bold">100만 건 처리 기준 30m → 1m으로 약 30배 이상</span>의 성능을 개선
  - ItemProcessor를 제거하여, 청크 단위 일괄 처리 구조로 개선
  - Paging 처리의 한계를 인지하고, Cursor 기반 Reader 적용
  - 배치 로직에서 JPA(ORM)을 걷어내고, Exposed DSL을 활용해 Batch Insert 적용
- 기존 AWS에서 운영되던 상용 환경 인프라를 교내 <span class = "bold">온프레미스 서버로 전환해 비용 100%</span> 감소
  - 온프레미스 환경에서 DB 데이터 안정성을 위해 1시간마다 덤프를 생성하는 백업 스크립트를 cron에 등록
- 서비스의 실시간 매트릭과 로그 수집, 대시보드 커스텀이 가능한 PLG를 채택하여 모니터링 시스템 구축

<span class="bold">V2 → V3 마이크로서비스 전환기</span>

- 단순한 기능 개선을 넘어, MSA 연구/학습 목적 + 목표 달성을 위해 마이크로서비스 설계를 진행함
  - V3의 간단한 기획 후 이벤트 스토밍 워크숍을 주최해 도메인을 분석하고, 총 7개의 MS를 도출함
  - Fallback, Retry, Circuit Breaker 패턴을 도입해 장애 전파 방지, 안정성을 높힘
  - 분산 트랜잭션 전략을 고민하고, Choreography 기반 SAGA 패턴을 적용
  - api-gateway에 JWT 인증 미들웨어를 개발해, 인증/인가를 중앙에서 처리
  - EDA로 도메인 간 결합도를 낮추고, 이벤트 기반 트랜잭션 처리, 보상 로직으로 데이터의 결과적 일관성을 유지함
  - 이벤트 유실에 대한 안정성과 높은 처리량을 고려하여 이벤트 큐는 Kafka를 채택함
